# Линтер для Progress 4GL
## Что он может делать
- Проверка keywords на case
- Указание конца блока для удобства чтения
- Отступы (ещё требуют реализации)
- Вывод warnings для определения сложных для оценки мест линтером (типа двойных и более комментариев в строке)

## Как используется
```
запуск
python main.py files [flags]

Опционально флаги:
--enc cp866 (cp866 или windows-1251 у меня в проекте Progress основные, иначе определяет самостоятельно по байтам)

--lines 850 (проходит по указанному количеству строк, если не указано - то весь файл)

--fix (Чтобы не только проверить но и исправить. Создает исправленную копию + файл .bak - для backup)

--no-display (Чтобы не выводить строчки, которые требуется исправить. Но выводит все сложные warnings)
```
